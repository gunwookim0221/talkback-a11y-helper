# UI 캡처 데이터셋 (UI Captures for Accessibility Testing)

## 목적
이 폴더는 SmartThings 앱에서 추출한 스크린샷(PNG), 계층 구조(XML), 메타데이터(JSON)를 모아둔 참조 데이터입니다.
주요 목적은 다음과 같습니다:
* **TalkBack 포커스 이동 분석**: 시각 장애인 사용자의 탐색 흐름 재현.
* **SMART_NEXT 동작 검토 및 수정**: 리팩토링된 탐색 로직의 정확도 검증.
* **구조 분류**: 상단바(App Bar), 콘텐츠 영역, 하단바(Bottom Navigation)의 정교한 분리.
* **스크롤 연속성 처리**: 스크롤 이후 새로 나타난 노드(Continuation)에 대한 포커스 정책 검토.

## 파일 명명 및 구성 원칙
파일은 관리 효율성과 AI 컨텍스트 제공을 위해 **평면 구조(Flat Structure)**로 관리하며 아래 규칙을 따릅니다.

### 파일명 형식
`{TabName}_{Index}_{State}.xml`

* **TabName**: 주요 탭 또는 화면 그룹 단위 (예: `devices`, `labs`, `settings`).
* **Index**: 00부터 시작하는 순차적 번호.
* **State (Suffix)**: 화면 상태 또는 스크롤 단계.
    * (없음): 일반 상태 또는 초기 진입 화면.
    * `scrolled`: 스크롤이 발생하여 화면 내용이 변화된 상태.
    * `bottom`: 리스트의 최하단에 도달하여 더 이상 스크롤이 불가능한 상태.

### 예시
* `devices_00.xml`: 기기 탭 진입 시 첫 화면.
* `devices_01_scrolled.xml`: 첫 번째 스크롤 발생 후 화면.
* `devices_02_bottom.xml`: 기기 탭의 마지막 최하단 영역.

## 구현 참고 원칙
이 데이터는 로직의 범용성을 확보하기 위한 구조 참고용이며, 특정 라벨을 하드코딩하기 위한 목적이 아닙니다.
* **범용 규칙 우선**: 상단 App bar 분류, 콘텐츠 경계 판별, 하단 네비게이션 제외 등은 좌표와 속성 기반의 범용 로직으로 처리해야 합니다.
* **예외 처리 지양**: `Labs`, `Devices`, `Menu` 등 특정 문자열에 의존하여 하드코딩된 예외 로직을 작성하지 않습니다.
* **스크롤 후보 처리**: 스크롤 후 새로 나타난 후보 노드를 우선적으로 탐색하는 정책을 검증합니다.

## 참고 사항
* **스크롤 분할**: 일부 긴 화면은 수동 스크롤을 통해 여러 Step으로 나누어 캡처되었습니다.
* **화면 성격 분류**: Settings 계열 등 하단 탭 루트와 성격이 다른 화면은 별도의 TabName으로 분리하여 관리합니다.
